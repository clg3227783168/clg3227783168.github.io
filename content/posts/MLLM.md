CLIP
学习判断一张图片和一段文字描述是否匹配。例如，它会学习到“一只在沙发上的猫”的图片特征与这段文本特征在语义空间中是接近的，而与“一辆飞驰的汽车”的文本特征是远离的

图像编码器：负责将图像转换为特征向量。通常采用 Vision Transformer (ViT)​ 或改进版的 ResNet​ 作为主干网络 。以ViT为例，它会将图像分割成小块，通过Transformer层处理，最终提取一个全局图像特征 。
文本编码器：负责将文本转换为特征向量。采用标准的 Transformer架构（如BERT），处理分词后的文本序列，并提取代表整个句子的语义特征 。
投影层与对齐：两个编码器输出的特征向量会分别通过一个投影层，被映射到同一个共享的嵌入空间。在这个空间里，语义相关的图像和文本特征会彼此靠近 。训练时使用对称的对比损失函数，目标是让匹配的图文对特征相似度最大化，不匹配的图文对特征相似度最小化 。



BILP
统一多模态理解和生成任务的预训练模型
三合一架构：BLIP的核心是一个多模态混合模型，它巧妙地共享参数，同时支持三种功能 ：
图文对比学习：使用单模态编码器，学习图像和文本的全局特征，并进行对齐。
图文匹配：使用基于图像的文本编码器，通过交叉注意力机制融合图像和文本信息，判断二者是否匹配。
图像描述生成：使用基于图像的文本解码器，以自回归的方式，根据图像内容生成描述文本 。
自举数据清洗：BLIP的一个关键创新是Captioner-Filter机制。模型先被微调成一个“描述生成器”和一个“噪声过滤器”，然后用它们来清洗和扩充有噪声的网络数据，再用这些高质量数据训练出更强大的模型，形成一个性能提升的良性循环 。



LLaVA
LLaVA的结构可以概括为“视觉感知，语言思考”。
视觉编码器：LLaVA本身不从头训练视觉部分，而是直接利用预训练好的CLIP模型的ViT图像编码器来提取图像的视觉特征 。
投影层：由于视觉特征和语言模型的词嵌入空间不一致，LLaVA引入了一个简单的投影层（通常是一个线性层或小型MLP），将视觉特征映射到语言模型能够理解的输入空间 。
大语言模型：处理文本和推理的核心是一个大语言模型。将投影后的视觉特征与文本提示（问题）一起输入LLaVA，大语言模型就能基于所“看到”的图像内容进行对话和推理 。

# 显存优化技术

## 训练过程中的显存占用的种类

1. 模型参数
2. 优化器状态
3. 中间激活值
4. 梯度
5. 显存buffer(用于kernel计算)
6. 显存碎片

## DP data parallel

把训练所需的所有参数(以上1~4)每个GPU放完整的一份，每个GPU分到同一batch的不同数据，都计算完之后进行All-Reduce：每个人报出一个数字，操作结束后，每个人都知道了所有数字的平均。

## ZeRO 1 切分优化器状态

将优化器状态分散到多个GPU
1. 类似DP，每个GPU能够独立进行前向传播和反向传播
2. All-Reduce，每个GPU得到整体梯度均值
3. 每个GPU计算自己持有那部分优化器状态对应的模型参数更新
4. All-Gather，每个GPU得到完整的模型参数

## ZeRO 2 切分优化器状态和梯度

1. 类似DP，每个GPU上都保存有完整的模型参数副本，能够独立进行前向传播
2. 一部分梯度计算完反向传播之后立马Reduce，将结果分片到每个GPU
3. 每个GPU使用自己持有的那部分优化器状态分片，再使用刚刚得到的梯度分片来更新模型参数（更新模型参数之后会更新优化器状态）
4. All-Gather，每个GPU得到完整的模型参数

## ZeRO 3 切分优化器状态和梯度和模型参数
每个GPU持有模型某一部分的模型参数、优化器状态和梯度，可以独立完成前向、反向传播和

## CPU Offload

优化器状态常驻CPU，在反向传播计算出梯度后，梯度被移动到CPU，优化器在CPU上更新参数，然后将更新后的参数分片移回GPU。

对模型每一层的层内参数进行切分，即对参数矩阵切片，并将不同切片放到不同 GPU 上。将原本在单卡中的矩阵乘法，切分到不同卡中进行矩阵乘法。

# 你只训练了投影层（Projector），这个模块的作用是什么？
投影层将视觉编码器输出的图像特征（Image Embeddings）映射到 LLM 的文本特征空间
# 投影层为什么要用 MLP 而不是线性层？
线性层假设视觉特征空间（CLIP）和文本特征空间（LLM）是线性相关的。
MLP引入了非线性能力,因为视觉特征空间到文本特征空间不是简单的线性变化可以得到的

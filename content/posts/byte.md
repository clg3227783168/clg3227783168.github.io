# 从客户端发出一个HTTP请求，一直到请求结束。整个过程当发生了什么
1. 客户端解析URL，提取协议、域名、端口、路径
2. 检查DNS缓存：如果存在，直接获取到目标IP地址，执行4；如果不存在，执行3
3. 发送DNS查询请求经由路由器到DNS服务器，查询目标IP地址
4. 客户端根据目标IP地址和服务端通过三次握手建立TCP连接
5. 客户端发送http请求
6. 通过反向代理将请求转发给具体的服务器
7. 服务器处理请求然后将处理结果封装成http响应报文
8. 客户端接收然后处理响应
9. 默认 HTTP/1.1 会保持连接复用，但一段时间无活动后，客户端会通过四次挥手关闭 TCP 连接

## 三次握手

三个缩写（SYN、ACK、FIN）指的是 TCP 报文首部（Header） 中的 控制位（Control Flags）。它们每一个只占 1 bit（1位），值为 1 代表有效，0 代表无效。

### 第一次
客户端发送SYN报文（Synchronize Sequence Numbers），该报文的SYN标志位=1，该报文中有一个随机的序列号x，作为初始序列号，数据将从这个序号开始。请求建立连接。

### 第二次
服务器收到客户端的SYN报文后，回发SYN+ACK 报文。SYN=1, ACK=1。确认号： Ack=x+1；随机生成自己的初始序列号y

### 第三次
客户端收到服务器的 SYN+ACK 报文后，向服务器发送一个 ACK 报文。标志位： ACK=1。序号： Seq=x+1。确认号： Ack=y+1。

## 四次挥手

### 第一次
客户端发送一个 FIN 报文，用来关闭客户端到服务器的数据传送。标志位： FIN=1。序号： Seq=u。

### 第二次
服务器收到 FIN 后，发送一个 ACK 给客户端。标志位： ACK=1。确认号： Ack=u+1。此时连接处于半关闭状态，客户端不再发送数据，但仍接收数据。

### 第三次
服务器的数据发送完毕后，发送一个 FIN 报文给客户端。标志位： FIN=1。序号： Seq=w。

### 第四次
客户端收到 FIN 后，发送一个 ACK 给服务器。标志位： ACK=1 确认号： Ack=w+1。

# 指针和引用的核心区别

## 概念
指针是一个独立变量，拥有自己的内存空间，用于存储另一个变量的地址
引用是某个已存在变量的别名，本身不占用独立存储空间

## 核心区别
指针可以被重新赋值；引用在初始化的时候会被赋值，不能为空，之后不能再修改值。

# C++ 多态的实现原理

## 运行时多态

### 概念
1. 核心机制：虚函数表（vtable）
当类中声明了虚函数（包括继承的虚函数），编译器会为该类生成一个虚函数表。
虚函数表是一个函数指针数组，按声明顺序存储该类所有虚函数的实际地址（指向最终调用的函数）。
每个类（不是每个对象）只有一个虚函数表，存储在静态数据区。
2. 每个对象中的隐藏指针：vptr
每个包含虚函数的类的对象中，编译器会自动添加一个隐藏的成员变量——虚函数表指针（vptr），通常放在对象内存布局的头部。
vptr指向该对象所属类的虚函数表。
当对象被构造时，构造函数会初始化 vptr，使其指向正确的虚函数表。
3. 动态绑定的过程
当通过基类指针或引用调用虚函数时：
程序通过对象的 vptr找到对应类的虚函数表。
在虚函数表中查找该虚函数的地址。
调用该地址指向的函数（可能是派生类重写的版本）。
### 要点
虚函数的绑定取决于对象的实际类型，而不是指针的声明类型

父类指针指向子类对象时，调用子类重写函数

#### 当使用子类指针指向父类对象时，调用虚函数会调用父类的版本，调用非虚函数则会调用子类的版本（这可能导致未定义行为）。​

```
class Base {
public:
    virtual void vFunc() { cout << "Base::vFunc\n"; } // 虚函数
    void nvFunc() { cout << "Base::nvFunc\n"; }       // 非虚函数
};

class Derived : public Base {
public:
    void vFunc() override { cout << "Derived::vFunc\n"; } // 重写虚函数
    void nvFunc() { cout << "Derived::nvFunc\n"; }        // 隐藏父类非虚函数（不推荐写法）
};
```

调用虚函数时，
```
Base baseObj;
Derived* pDerived = (Derived*)&baseObj; // 危险！强制转换让子类指针指向父类对象
pDerived->vFunc(); // 输出什么？
```
baseObj是一个纯粹的 Base类对象，它的虚函数表指针 vptr指向的是 Base类的虚函数表。
调用 vFunc()时，程序会通过这个 vptr去查找虚函数表，并调用其中存储的函数地址。
由于对象的“本质”是 Base，所以它的虚函数表里 vFunc项指向的就是 Base::vFunc。

调用非虚函数时，
```
pDerived->nvFunc(); // 输出什么？
```
非虚函数（nvFunc）的调用在编译期就根据指针的静态类型（即声明类型）确定。
指针 pDerived的静态类型是 Derived*，所以编译器会直接去查找 Derived::nvFunc的地址并调用。
巨大风险：如果 Derived::nvFunc试图访问任何 Derived类特有而 Base类没有的成员变量或函数，程序将访问到未知的内存区域，导致未定义行为（崩溃或数据错误）。因为实际的对象 baseObj根本没有那些成员的内存空间。

## 缓存

### CDN content distribute network
将内容提前分发到离用户最近的网络节点。当用户请求资源时，将请求路由到最近节点

### 如果数据不存在，什么时候去更新缓存
1 旁路缓存 （即数据不一致的解决办法）
2 当缓存失效的时候，通过分布式锁，只允许一个线程查询数据库并写入缓存 （即缓存击穿的解决办法）
当某个请求发现缓存未命中时，不立即查询数据库，先获取一个与key相关的分布式锁，再访问数据库；没有获得锁的短暂休眠后重试或者失败回退
3 异步更新缓存
当发现缓存过期时，直接返回旧数据；并单开一个线程进行缓存的更新。优点是性能高，用户不会感知到延迟；缺点是短时间内读到了旧数据，适合一致性不高的场景
抖音推荐视频场景用户的行为是浏览，而不是精确获取，流畅获取下一个内容，比使用最新的用户画像和特征以及最新点赞收藏数更重要。用户的兴趣视频本来就是一个兴趣池，而非精准无误的答案。

### 使用缓存技术对稳定性或延迟等指标的风险？

#### 数据不一致
数据库更新但是缓存没有成功写入会导致用户读到脏数据
解决办法：写操作只删缓存，不更新缓存。对“读数据库+写缓存”这一步加锁

读数据库流程
1. 收到读请求。
2. 首先，从缓存（如Redis）中读取数据。
   - 如果缓存命中（Cache Hit）：直接返回数据，流程结束。
   - 如果缓存未命中（Cache Miss）：
        a. 从数据库中查询数据。
        b. 将查询到的数据写入缓存（以便后续请求命中）。
        c. 返回数据。
写数据库流程
1. 收到更新数据的请求。
2. 直接更新数据库（完成持久化）。
3. 删除缓存中对应的Key。

#### 缓存穿透
数据在缓存和数据中都不存在，导致数据库压力很大

解决办法：
1：使用布隆过滤器来过滤非法请求
布隆过滤器使用1个bit来存储某个数据存在与否。元素加入过滤器时，使用哈希函数（可以有多个）对元素值进行计算，得到哈希值，把数组的哈希值对应的下标的位置置为1
判断元素是否存在时，哈希计算后判断数组的对应位置值是否都为1
2. 将空结果写入缓存，而不写入数据库，而且设置一个较短的过期时间。因为缓存的查询更高效。

#### 缓存击穿
热点的key并发请求，如果此时热点的key刚好过期，导致请求去往数据库，造成数据库瞬时压力激增。

解决办法：
当缓存失效的时候，通过分布式锁，只允许一个线程查询数据库并写入缓存

#### 缓存雪崩
大量缓存同时过期了
解决办法：设置随机的过期时间

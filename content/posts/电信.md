## 通信机制
### 内存映射
GPU的显存也是整个内存空间的一部分,像内存条一样
### DMA
CPU不作为GPU和内存两地之间的搬运工,而是GPU和内存之间通信
GPU向DMA控制器发送传输请求,DMA控制器向CPU发出总线占用请求,占用结束了之后通知CPU自己占用结束了.

#### 应用程序触发硬件操作,硬件操作完通知应用程序的方式
1. 轮询 在此期间,应用程序始终占有CPU,不断检查状态位
2. 中断 硬件通过中断通知CPU操作完成,CPU和应用程序之间关于知晓硬件操作已完成通过"内核通知机制"

##### 内核通知机制
建立两个应用和程序共享的环形队列,提交和完成
应用程序将IO请求放入提交队列,然后继续执行
内核异步处理请求
请求完成后,将完成事件放入完成队列
应用可以通过轮询直接检查完成队列,或者由内核通知应用程序(具体实现为epoll)

# 实现一个分布式缓存系统
## 要求
### 最终一致性
写操作先在本地结点完成,再异步广播到所有结点
### 全局可访问
用户无论资源的具体存放地址,使用一种访问方式
使用一致性哈希实现

#### 一致性哈希
hash(key)%2^32
增删结点只影响该结点顺时针后继的结点
##### 使用场景
hash(key)%n,当n改变的时候需要迁移数据,如何避免过多数据迁移

##### 均衡度
增加哈希结点的数量,数量越多越均衡
如果没有那么多结点,就让一个实际结点对应多个哈希结点
这样的话当移除一个实际结点的时候,移除了许多个虚拟结点,这些虚拟结点的后继结点可能对应着很多实际结点

### 用户进程的崩溃不影响系统稳定性
解耦,将数据存储,同步,一致性维护等放在一个守护进程中,用户进程只负责get/set

# 题目：Master 中心管控的分布式引用计数（带事件监听 & Worker 退出资源回收）
背景
在分布式系统中，多个 Worker 节点会创建和使用共享对象（目前只考虑是抽象的资源，具体的可以举例如某个版本的key-value对），每个对象都有唯一 ID。系统采用 Master 中心化管理引用计数：
- Master 维护对象的全局引用计数。
- Worker 通过向 Master 发送 RPC 请求增加或减少对象引用计数。
- 如果 Worker 节点退出，Master 也需要做对应的处理。
- Rpc 包返回要考虑下错误返回格式
加分考虑项：
- 大量对象情况下可以怎么优化
- 实际场景中，master如何正确监听到worker退出
- Worker 退出后再启动发起请求，master能否正确处理，怎么做到

说明：
- AcquireRequest 对应 Worker 请求增加对象引用计数。
- ReleaseRequest 对应 Worker 请求减少对象引用计数。
- RefCountResponse 返回更新后的全局引用计数。
- WorkerExit Worker 退出事件。

---
2. Master 节点接口
需要考虑并发安全
class Master {
public:
    virtual ~Master() = default;

    // Worker 请求增加引用计数
    virtual RefCountResponse handle_acquire(const AcquireRequest& req) = 0;

    // Worker 请求减少引用计数
    virtual RefCountResponse handle_release(const ReleaseRequest& req) = 0;

    // Worker 节点退出事件
    virtual void on_worker_exit(const std::string& worker_id) = 0;

    // 事件监听接口
    using CollectCallback = std::function<void(const WorkerExit&)>;
    virtual void set_collect_callback(CollectCallback cb) = 0;
    
    // 内部触发通知接口（Worker 内部可能通过 RPC 实现）
    virtual void notify_worker_refcount_change(const std::string& worker_id,
                                               const RefCountNotification& note) = 0;
};

---
3. Worker 节点接口
class Worker {
public:
    virtual ~Worker() = default;

    // 向 Master 发送 AcquireRequest，返回更新后的全局引用计数
    virtual RefCountResponse acquire(const std::string& object_id) = 0;

    // 向 Master 发送 ReleaseRequest，返回更新后的全局引用计数
    virtual RefCountResponse release(const std::string& object_id) = 0;
    
    // 统一处理 Master 通知
    virtual void handle_refcount_notification(const RefCountNotification& note) = 0;
};
ManagedObject (RAII)
class ManagedObject {
public:
    ManagedObject(Worker* worker, const std::string& object_id);
    ~ManagedObject();

    ManagedObject(const ManagedObject&) = delete;
    ManagedObject& operator=(const ManagedObject&) = delete;

    // 支持作用域管理
    ManagedObject(ManagedObject&& other) noexcept;
    ManagedObject& operator=(ManagedObject&& other) noexcept;

private:
    Worker* worker_;
    std::string object_id_;
};
